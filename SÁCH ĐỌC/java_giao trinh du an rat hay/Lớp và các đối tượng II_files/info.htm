Lớp và các ðối tượng II

Tiếp theo bài giảng trước, chúng ta sẽ xem xét kỹ hơn cấu hình của lớp và ðối tượng. Bên cạnh ðó bài giảng sẽ hướng dẫn thêm một vài vấn ðề cơ bản của thực tế tập trình bằng ngôn ngữ Java. 

1. Truy cập các thành viên trong lớp

1.1. Phân tích cụ thể

Lúc này chúng ta xem xét lớp Pair một cách chi tiết hơn.  

class Pair { 

    int a;    // miền của lớp xác ðịnh các thành phần ðối tượng
    int b;    // a – thành phần ðầu tiên, b – thành phần thứ hai

 Pair(int x, int y) {   // the constructor thiết lập giá trị của các thành phần
   a = x  ;             // dựa vào các giá trị làm ðối số 
   b = y;
 }
 
 Pair(int x) {          // constructor thiết lập giá trị của các thành phần
   a = b = x;           // (a và b) cùng bã̀ng giá trị x
 } 


 void set(Pair p)  {    // phương pháp này thiết lập giá trị của các thành phần
   a = p.a;             // dựa trên các thành phần của cạ̃p cung cấp làm ðối số 
   b = p.b;
 } 


 Pair add(Pair p) {             // phương pháp cộng hai cạ̃p
   Pair result = new Pair(a, b);
   result.a += p.a;
   result.b += p.b;
   return result;
 }
 
                               // phương pháp in một cạ̃p
 void show(String s)  {                 
   System.out.println(s + " ( " + a + " , " + b + " )" );
 }

}


Chúng ta có thể sử dụng lớp Pair trong một lớp khác như sau:

class PairTest {
 
  public static void main(String[] args) {
    Pair pair1 = new Pair(1,5);
    Pair pair2 = new Pair(2,4);
    pair1.show("Pair 1 =");    
    pair2.show("Pair 2 =");
    Pair pairSum = pair1.add(pair2);
    pairSum.show("Sum of pairs =");
    pair1.set(pair2);
    pair1.show("Pair 1 = ");
 
  } 
 
}

Kết quả:
 
Pair 1 = ( 1 , 5 )
Pair 2 = ( 2 , 4 )
Sum of pairs = ( 3 , 9 )
Pair 1 =  ( 2 , 4 )

    •    ðịnh vị vùng bộ nhớ ðể lưu ðối tượng (hai số nguyên)

    •    giá trị của biểu thức new là một tham chiếu ðến ðối tượng vừa ðược tạo ra

        •    mặc dù ðôi khi chúng ta nói về các ðối tượng pair1 và pair2 (ðể ðơn giản hóa vãn bản) nhưng chúng trên thực tế là các biến giữ tham chiếu ðến ðối tượng.

Quy tắc tương tự ðược áp dụng với việc gọi phương pháp cho ðối tượng.

Sau ðây là thuật toán ðể thiết lập

    1.    giá trị của miền a của ðối tượng ðược ðưa vào với tư cách là ðối số ðược lưu vào miền a của ðối tượng mà phương pháp ðã gọi
    2.    giá trị của miền b của ðối tượng ðược ðưa vào với tư cách là ðối số ðược lưu trong miền b của ðối tượng mà phương pháp ðã gọi

Ðối tượng mà phương pháp ðã gọi ðược thể hiện bã̀ng từ khóa this bên trong phương pháp.

   b = p.b;

Trong một số tình huống từ khóa this cần thiết phải có. Ví dụ, nếu tên của các tham số giống tên của các miền thì:

class Pair {
    int a, b;

    Pair (int a,  int b) {
        a = ... // a nào: tham số hay miền?
        ...
    }

    void set(int a, int b) {
        a = ... // a nào: tham số hay miền?
    }
}


this.a luôn biểu thị miền a của ðối tượng ðược khởi tạo bởi constructor hay ðối tượng mà phương pháp ðược gọi. 
class Pair {
    int a;
    int b;

    Pair(int a, int b) {

        this.a = a;    // lưu giá trị của tham số a vào miền a
        this.b = b;    // lưu giá trị của tham số b vào miền b
    }

    void set(int a, int b) {

        this.a = a;
        this.b = b;
    }
}

1.2. Các thành viên tĩnh

Các thành viên là các miền và phương pháp của lớp.
Các miền và phương pháp của lớp có thể tĩnh hoạ̃c ðộng (non-static). Cho ðến giờ chúng ta mới chỉ dùng các thành viên ðộng.

Các thành viên ðộng luôn kết nối với một ðối tượng ðang tồn tại (các miền thể hiện dữ liệu ðược lưu trong ðối tượng, các phương pháp là các thông ðiệp ðược chuyển vào ðối tượng và phải ðược gọi cho ðối tượng).


Các thành viên tĩnh (miền và phương pháp):

    •    ðược khai báo với từ bổ sung static
    •    có thể ðược sử dụng thậm chí khi không có ðối tượng nào trong lớp tồn tại


Các thành viên tĩnh có thể ðược truy cập bởi tên của lớp kèm theo dấu chấm và tên miền:
tên_lớp.tên_miền

Trong hệ thống các lớp (có trong J2SDK) có phương pháp tĩnh là exit(int) ðể ngã́t việc chạy ứng dụng (và trả lại mã code return).

Do ðó, ðể ngã́t chương trình với mã thoát 0 chúng ta viết:

System.exit(0);

Lớp system có một miền tĩnh có tên là out quy ðịnh dữ liệu ðầu ra chuẩn của chương trình (bảng nhã́c câu lệnh). Ðối tượng out này hiểu thông ðiệp println. Thông ðiệp này dẫn ðến việc in ðối số ở dạng chuỗi ra bảng nhã́c câu lệnh (println là một phương pháp của lớp PrintStrean; out la ðối tượng của lớp này).

Vì thế chúng ta viết: System.out.println(...)..

Chỉ có các biến thể là miền của lớp mới có thể ðược khai báo static.

Việc nhận ra sự khác biết giữa các thành phần tĩnh và ðộng là hết sức quan trọng. Bởi các thành phần tĩnh miêu tả thuộc tính của lớp chứ không phải ðối tượng của lớp.

Cụ thể ðối với các miền sự khác biệt ðó là như sau:

    •    các miền ðộng tồn tại (ở dạng bộ phận cấu thành) trong mỗi ðối tượng của lớp
    •    các miền tính không nã̀m trong ðối tượng; mỗi biến thể tĩnh chiếm một ô trong bộ nhớ cho toàn bộ lớp ðối tượng.

Ví dụ, trong một lớp mô tả các ổ cứng chúng ta có một miền tĩnh tên là vat và các miền ðộng: model (kiểu), dung lượng, giá cả. Các ổ cứng khác nhau về kiểu, dung lượng và giá nhưng có cùng một giá trị VAT, do ðó không cần phải lưu giá trị này trong mỗi ðối tượng riêng biệt. Ðây là một thuộc tính chúng của tất cả các ổ ðĩa. Là một biến thể tĩnh nên nó chỉ chiếm 8 byte trong bộ nhớ, bất kể số lượng ðĩa (ðối tượng) ðược tạo ra trong chương trình là bao nhiêu..
public class Disk  {

  private static double vat;

  private String model;
  private int capacity;
  private double price;
  
  public Disk(String m, int c, double p) {
    model = m;
    capacity = c;
    price = p;
  }

  public String getDescription() {
    return model + " ," + capacity + " MB";  
  }
  
  public double getGrossPrice() {
    return price * ( 1 +  vat/100 );
  }

  public static void setVat(double v) {
    vat = v;
  }
 
}

class Test {
   
  public static void main(String[] args) {
    Disk.setVat(22.0);
    Disk d1 = new Disk("IBM598", 6040, 430.0);
    System.out.println(d1.getDescription() + " - price " 
                       + d1.getGrossPrice() + " EUR");
  }
}
IBM598 ,6040 MB - price 524.6 EUR
Chú ý rã̀ng phương pháp setVat ðược gọi trước khi bất cứ một ðối tượng nào ðược tạo ra. Nó sẽ thiết lập giá trị của miền vat dù chưa có ðối tượng nào của lớp Disk tồn tại.

Các phương pháp tĩnh không thể truy cập vào các thành viên ðộng của lớp. Có thể truy cập vào các thành viên tĩnh khác.


Các phương pháp ðộng ðương nhiên có thể truy cập vào các thành viên tĩnh theo cách thông thường (xem phương pháp getGrossPrice()).

2. Phạm vi của từ ðịnh danh. Các biến cục bộ. Kiểm soát truy cập


Phạm vi của một từ ðịnh danh là vùng chương trình mà trong ðó nó có thể ðược quy chiếu ðến bằng cách sử dụng một tên ðơn giản.


Các thành phần của lớp (miền và phương pháp) có thể ðược quy chiếu từ bất của phương pháp nào của lớp bất kể khai báo của thành phần ðó xảy ra ở ðâu trong cấu trúc lớp.

Mỗi phương pháp có thể khai báo các biến của riêng nó (hay các hằng số). Pham vi của các từ ðịnh danh mang tính cục bộ - nghĩa là nó bắt ðầu ở nơi khai báo và kết thúc tại cấu trúc phương pháp (tại dấu ngoặc ðóng của phương pháp). Những biến như thế gọi là biến cục bộ. Các tham số phương pháp có cùng một phạm vi. Trên thực tế các tham số là các biến thể cục bộ.

Xem xét lớp sau::
class A {

    int a;

    void method1() {
        int b;
        ...   
    }

    void method2() {
        int c;
        ...
    }
}


Bên trong phương pháp method1 chúng ta có thể truy cập vào các biến thể a và b và phương pháp method2. Bên trong phương pháp method2 chúng ta có thể truy cập và biến thể a và c và phương pháp method1. Biến thể c không thể ðược quy chiếu ðến bên trong phương pháp method1, và tương tự biến thể b không thể ðược quy chiếu ðến bên trong phương pháp method2.


Bên trong một constructor hay một phương pháp, một từ ðịnh danh miền của lớp có thể ðược che ði.

Ví dụ:

class A {

    int a;

    void method() {

        int a = 0;     // the field a is shadowed by the local variable a
        a = a + 10;    // the local variable a is used here
        this.a++;      // the class field is used here
    }
}

Biến cục bộ x ðược khai báo bên trong phương pháp method(). Nó có tên giống tên miền của lớp, do ðó biến cục bộ truy ra miền của lớp. Biểu thức a trong phương pháp() quy chiếu ðến (biểu thị) giá trị của biến thể cục bộ. Miền lớp có thể ðược truy cập bằng cách sử dụng tên ðạt tiêu chuẩn: this.a.

Trong Java các biến cục bộ không thể ðược truy ra trong khối cục bộ.


Do ðó ðịnh nghĩa:

class A {
 ....
  void method() {
     int a;
     {
       int a;
         ...
     }
  }
}

là sai.


Khác với các biến thể của lớp (các miền), biến thể cục bộ không tự ðộng ðược khởi tạo. Giá trị của biến thể cục bộ không ðược xác ðược cho ðến khi nó ðược khởi tạo bởi một bộ khởi tạo hay một chỉ ðịnh.

 
Các biến thể cục bộ phải ðược khởi tạo một cách rõ ràng.


Còn vấn ðề truy cập các thành viên khác của lớp thì sao?

Việc này ðược quyết ðịnh bởi các từ bổ sung truy cập.

Khả nãng truy cập của một thành viên lớp có thể là:

    •    private (riêng) – cho phép truy cập từ bên trong lớp mà nó ðược khai báo (từ bổ sung private)
    •    default (mặc ðịnh)– cho phép truy cập từ trong lớp xác ðịnh trong cùng gói (không yêu cầu từ bổ sung)
    •    protected (bảo vệ) – thành viên cho phép truy cập từ trong lớp, và tất cả các lớp con và các lớp xác ðịnh trong cùng một gói (từ bổ sung protected)
    •    public (chung)– thành viên luôn cho phép truy cập (từ bổ sung public)

Chúng ta ðã tìm hiểu một số lý do của việc sử dụng từ bổ sung truy cập. Chú ý rằng những từ này giới hạn việc truy cập vào các thành viên trong lớp:

import javax.swing.*;

public class Pair {

    private int a;
    private int b;
    public String name;

    public Pair(int x, int y) {
        a = x;
	b = y;
    }

    private String makeString() {
        return name + " " + a + " and " + b;
    }

    public void show() {
        JOptionPane.showMessageDialog(null, makeString()); // 6
    }
}

class Test {

    public static void main(String[] args) {

        Pair p = new Pair(17,20);
        p.name = "Pair of numbers";           // 1
	p.a = 1;                              // 3
	System.out.println(p.makeString());   // 4
	p.show();                             // 5
    }
}

Biên dịch sinh ra một số lỗi sau:

Pair.java:29: a has private access in Pair
    p.a = 1;                              // 3
     ^
Pair.java:30: makeString() has private access in Pair
    System.out.println(p.makeString());   // 4
                        ^
2 errors

Chú ý:

    •    dòng biểu thị bởi //1 ðúng vì miền name ðược khai báo public
    •    dòng biểu thị bởi //3 và //4 sinh ra các lỗi khi các miền ðược khai báo private không cho phép truy cập từ lớp khác
    •    dòng biểu thị bởi //5 ðúng vì chúng ta truy cập ðược phương pháp chung
    •    dòng biểu thị bởi //6 ðúng vì chúng ta truy cập ðược các thành viên private từ trong lớp mà thành viên ðược xác ðịnh.

Các lớp có thể ðược khai báo public. Những lớp ðó cho phép bất cứ mã nào truy cập. Nếu một lớp không ðược khai báo public thì nó chỉ cho phép truy cập từ trong nhóm mà nó ðược khai báo.

Ðể khai báo một lớp chung phải sử dụng từ bổ sung public.
 
public class Pair {
 ...

}

Một tệp nguồn có thể chứa nhiều nhất một lớp chung. Tên của lớp này phải giống hệt như tên của file.

3. Khái niệm về tính kế thừa 


Sự kế thừa cho phép sử dụng lại chức nãng và thuộc tính của một lớp hiện tại bởi một một lớp khác với một số sửa ðổi khả thi mà sẽ làm cho lớp mới trở nên cụ thể hơn.


Lớp Publication thể hiện các ấn phẩn nhưng nó không thể hiện các cuốn sách một cách ðầy ðủ. Các quyển sách là loại ấn phẩm ðược cụ thể hóa. Ngoài những thuộc tính của một ấn phẩm ra (như tiêu ðể, nhà xuất bản, giá cả vân vân), một cuốn sách còn có thêm một thuộc tính ðó là: tác giả (hoặc các tác giả).
Do ðó ðể thể hiện việc mua và bán của các cuốn sách chúng ta cần có một lớp thể hiện chúng. Chúng ta hãy gọi lớp ðó là Book.
Chúng ta có thể xác ðịnh lớp Book từ những thông tin như (xác ðịnh các miền: tác giả, tiêu ðề, giá cả và phương pháp thao tác trên các miền cũng như các phương pháp ðể mua và bán chúng).
Tuy nhiên lớp Publication cũng cung cấp hầu hết các thuộc tính yêu cầu và phương pháp, do ðó tố hơn chúng ta nên kế thừa lớp Book từ lớp Publication. Dĩ nhiên chúng ta phải thêm những thuộc tính của sách không có trong lớp Publication.



Từ khóa extends (mở rộng) thể hiện sự kế thừa của các lớp:

Chúng ta viết:

                class B extends A {
                    ...
                }

có nghĩa là lớp B kế thừa (ðược trích ra) từ lớp A.

Chúng ta nói:

    •    Lớp A là lớp cấp trên (lớp nền) của lớp B
    •    lớp B là lớp cấp dưới của lớp A


Chúng ta hãy xác ðịnh lớp Book. Bên cạnh những thuộc tính của lớp cấp trên Publication chúng ta cần một miền lưu tác giả của cuốn sách và một phương pháp ðể trả lại giá trị của nó.
class Book extends Publication {
    private String author;

    public String getAuthor() {
        return author;
    }
}

Liệu như vậy ðã ðủ chưa?

Chúng ta còn cần một constructor mô tả về các miền hợp trong lớp Book ðược khởi tạo như thế nào.

Ðể tạo ra một ðối tượng của lớp này chúng ta phải cung cấp các thông tin sau:

    •    tác giả
    •    ðầu ðề
    •    nhà xuất bản
    •    nãm xuất bản
    •    ký hiệu sách
    •    giá cả
    •    số sách có trong kho

Do vậy constructor sẽ có dạng sau:

 public Book(String aut, String tit, String pub, int y, String id,
             double price, int quant) {
 ....
 }
Nhưng thật không may, các miền như: tiêu ðề, nhà xuất bản, nãm xuất bản, ký hiệu sách, giá và số lượng lại có truy cập private trong lớp Publication, do vậy chúng ta không thể có những thông tin này. Làm thế nào ðể khởi tạo chúng ðược?

Các miền của lớp cấp trên ðược khởi tạo với việc gọi constructor của lớp cơ bản từ constructor của lớp phát sinh.


Câu lệnh sau:

        super(danh_sách_tham_số);

là một lệnh gọi constructor từ lớp cấp trên với các ðối số ðã cho trong danh sách tham số.

Nếu constructor tồn tại, câu lệnh như thế phải là câu lệnh ðầu tên của constructor trong lớp phát sinh.
Còn nếu không tồn tại thì constructor mặc ðịnh của lớp cơ bản sẽ ðược gọi trước khi tạo ra ðối tượng của lớp cấp dưới.


Constructor của lớp Book phải gọi constructor của lớp cấp cao ðể khởi tạo các miền của nó và sau ðó khởi tạo miền tác giả.

  // arguments: aut - author, tit - title, pub - publisher, y - year
  //            id - ISBN, price - price, quant - quantity   
  public Book(String aut, String tit, String pub, int y, String id, 
              double price, int quant) {
    super(tit, pub, y, id, price, quant);
    author = aut;
  }

 public String getAuthor() {

 }

}

4. Gói và nhập khẩu

4.1. Khái niệm gói


Gói là một loại tập hợp lớp nhóm các lớp có chức nãng tương tự lại với nhau như: các thao tác ðầu vào – ðầu ra, các ứng dụng trong mạng, vân vân.

Mỗi lớp thuộc về một gói nào ðó.
Các lớp ðược biên dịch với khai báo package… thuộc về gói khai báo.
Các lớp ðược biên dịch mà không có khai báo gói thuộc về gói mặc ðịnh (không tên).

Các gói có trật tự từ cao ðến thấp:

aPackage
  aSubPackage
    aSubPackage

Ví dụ, gói java.awt chứa gói phụ là java.awt.event.

Trật tự cấp bậc của các gói ðược sắp xếp theo cấu trúc thư mục (các cấp ðộ liên tiếp của thư mục trong trật tự thư mục tương ứng với cấp ðộ liên tục trong trật tự gói).

Vậy chúng ta lại sử dụng các gói ðể làm gì?

Xem xét ví dụ sau:

class A {
    Button b = new Button("Ok");  // new object of the class Button
....
}

Có khả nãng hay trường hợp:
   
    1.    lớp Button ðược xác ðịnh trong cùng một tệp (như lớp A) hay trong một tệp khác của thư mục

    2.    lớp Button ðược xác ðịnh ở một nơi khác

Trong trường hợp ðầu tiên trình biên dịch (và bộ máy ảo Java) có thể tìm ra lớp Button.

Trong trường hợp thứ hai một vấn ðề phát sinh là: tìm lớp Button ở ðâu?

Hơn nữa, việc sử dụng tên Button có thể gây mơ hồ (liệu ðó có phải Button ở thư mục hiện tại hay ở một nơi nào khác?).

Do ðó các gói nhóm các lớp lại vào một vùng và bảo vệ khỏi sự xung ðột về tên.

Tên chuẩn:

Tên chuẩn của một lớp (loại) thuộc về một gói tên có dạng sau:

package_name.class_name

Ví dụ:   java.awt.Button

class A {
     java.awt.Button b = new java.awt.Button("Ok");
     ....
}

Ðối với các lớp xác ðịnh trong gói mặc ðịnh (không tên) thì tên ðủ tiêu chuẩn là tên giống tên của lớp (ví dụ: Pair).

4.2. Nhập khẩu


Thông thường tất cả các tên lớp sử dụng trong chương trình cầ̀n phải ðạt tiêu chuẩn.

May thay câu lệnh import cho phép sử dụng các tên ðơn giản.
import java.awt.Button;  // nhập tên của lớp java.awt.Button 

class A { 
 java.awt.Frame f = new java.awt.Frame("Title"); 
 Button b = new Button("Ok");  // cách dụng tên ðơn giản của lớp java.awt.Button 
.... 
}

Bởi vì tên của lớp java.awt.Button ðược nhập khẩu nên chúng ta có thể sử dụng tên giản lược của lớp trong chương trình. Nhưng tên của lớp java.awt.Frame không ðược nhập khẩu về nên chúng ta phải sử dụng dạng tên chuẩn ðầy ðủ của nó.

Ðể nhập tất cả các tên của lớp từ một gói ðã cho chúng ta phải sử dụng dấu sao:

import java.awt.*;     // nhập khẩu tát cả các tên lớp từ gói java.awt 
class A { 
     Frame f =  new Frame("Title");     // gời các tên giản lược có thể ðược dùng 
     Button b = new Button("Ok"); 
.... 
}


Các lớp String và System ðược sử dụng thường xuyên trong các chương trình là thuộc về gói java.lang.

Gói java.lang không yêu cầu câu lệnh nhập khẩu.
Các lớp từ gói này ðược nhập khẩu một cách tự ðộng.


4.3. Nhập khẩu tĩnh

Nhập khẩu tĩnh chio phép gọi ðến các thành phần tĩnh bằng tên ðơn giản.



Nhập khẩu tĩnh:

import static TypeName.Identifier;


Nhập khẩu tất cả các thành phần tĩnh:

import static TypeName.*;



Ví dụ:

import static java.lang.System.out;

public class MyClass {

  public static void main(String[] args) {
    out.println("Ngắn hơn!");
  }

}
hoặc:
import static java.lang.System.*;

public class MyClass {

  public static void main(String[] args) {
    long time = nanoTime();  // System.nanoTime()
    out.println(time);       // System.out
    exit(1);                 // System.exit(..)
  }

}

6. Thực hành: một số ðối tượng và phương pháp hay dùng

6.1. Chuỗi ký tự

Các chuỗi (ký tự) là ðối tượng của lớp String. Do ðó tất cả những gì ðược nói về ðối tượng và tham chiếu cũng thuộc về ðối tượng của lớp String.
Tuy nhiên, các thao tác trên chuỗi rất thông dụng cho nên ðể làm cho việc lập trình trở nên dễ dàng hơn trình phiên dịch giúp cho việc xử lý các chuỗi ðược thuận tiện hơn theo nhiều cách. Tuy nhiên ðiều này cũng gây ra một số lúng túng. Các ðối tượng của lớp String có thể ðược tạo ra theo cách tương tự như các ðối tượng của các lớp khác.

Nhưng trình phiên dịch ðã giúp chúng ta với mẹo sau:

thay vì viết

String = new String (“Abc”);

chúng ta có thể viết:

String = “Abc”;


Kết quả là:

    •    Một ðối tượng mới của lớp String thể hiện chuỗi “Abc” ðược tạo ra và
   
    •    một tham chiếu ðến ðối tượng này ðược lưu vào trong biến thể s.

Do ðó ðể tạo ra các ðối tượng của lớp String chúng ta không cần phải sử dụng biểu thức new.



Một phương tiện khác ðược cung cấp bởi trình phiên dịch là toán tử + ðược sử dụng ðể kết nối chuỗi. Ví dụ:
String s1 = "All work and no play";
String s2 = " makes Jack a dull boy";
String s3;
s3 = s1 + s2;

Hoạt ðộng như sau:

    •    biểu thức s1+s2 tạo ra một ðối tượng mới của lớp String thể hiện sự kết nối của các chuỗi biểu thị bởi s1 và s2.

    •    một tham chiếu ðến ðối tượng vừa tạo ra ðược lưu trong biến thể s3.

    •    s3 thể hiện chuỗi “All work and no play makes Jack a dull boy”.

Hơn nữa, toán tử + cho phép ðính kèm các dữ liệu thuộc kiểu khác vào các chuỗi (số hay thậm chí là các ðối tượng). Ví dụ:

String s1 = "Jack ";
String s2 = "is ";
String s3;
s3 = s1 + s2 + 3 + " years old";

Giờ ðây biến thể s3 thể hiện chuỗi “Jack is 3 years old”.

Dĩ nhiên chúng ta có thể sử dụng biến kiểu int thay vì dãy 3.

int years = 3;
...
s3 = s1 + s2 + years + " years old";

Biến thể year cũng như dãy 3 trong biểu thức nối là biến thể thuộc kiểu int. Các giá trị số khác (kiểu float, double và các kiểu khác) cũng có thể ðược sử dụng.
Nếu tác tử bên phải của toán tử + là một tham chiếu ðến một ðối tượng của một lớp nào ðó thì ðối tượng này ðược chuyển ðổi sang một chuỗi và ðược ðính kèm vào tác tử bên trái (một chuỗi). Lại một lần nữa chúng ta bắt gặp một mẹo của trình phiên dịch, nhưng sự thể hiện chuỗi của một ðối tượng ðược xác ðịnh bởi lớp của ðối tượng. Chúng ta sẽ tìm hiểu làm thế nào ðể có ðược và làm thế nào ðể sử dụng những mẹo như thế sau này.

Ví dụ chúng ta xem 2 dòng lệnh sau:

Date now = new Date();
String s = “Now is:” + now;

Ở ðây một ðối tượng của lớp Date ðược tạo ra. Nó lưu ngày hiện tại và thời gian ở dang nhị phân. Sauk hi kết nối các biến thể quy chiếu ðến một chuỗi miêu tả ngày và thời gian hiện tại: “Now is: Mon Jul 01 05:55:25 CEST 2008”


Các thao tác trên chuỗi liên quan ðến các tham chiếu cho nên cần ðặc chú ý khi so sánh:


Toán tử ðẳng thức (== và !=) áp dụng cho các biến thể biểu thị ðối tượng so sánh các tham chiếu chứ không phải bản thân các ðối tượng.


Do vậy chương trình sau:
String s1 = "Jo";
String s2 = "hn";
String s3 = s1 + s2;
String s4 = "John";
System.out.println(s3 + " " + s4);
if (s3 == s4) System.out.println("It is John");
else System.out.println("It is not John");

in ra (ngược lại so với những gì chúng ta nghĩ):

John John
It is not John

Tại sao:

    •    Ðối tượng biểu thị bởi s3 thể hiện chuỗi “John”.

    •    Tương tự, ðối tượng biểu thị bởi s4 thể hiện một chuỗi khác cũng là “John”.

    •    Nhưng kết qủa của s3=s4 là sai vì s3 và s4 quy chiếu ðến các ðối tượng khác nhau (mặc dù chúng thể hiện cùng một chuỗi).
   
    •    Tuy nhiên kết quả sẽ là ðúng nếu s3 và s4 cùng quy chiếu ðến một ðối tượng.


Không nên so sánh các chuỗi bằng các toán tử ðẳng thức  như == và !=.

Thay vào ðó sử dụng phương pháp equals thuộc lớp String.

Nội dung của hai chuỗi s1 và s2 có thể ðược kiểm tra về tính bình ðẳng bằng phương pháp gọi hàm:

s1.equals(s2)

Nội dung của chuỗi s và dãy chuỗi có thể ðược so sánh theo cách tương tự:

s.equal(a_string_literal)

Nếu cả hai chuỗi ðều chứa cùng một chuối ký tự, kết quả sẽ cho là true, nếu không kêt quả là false.

Tuy nhiên, nên biết rằng:
Tất cả các dãy chuỗi thể hiện cùng một vãn bản quy chiếu ðến một ðối tượng. Câu lệnh aString = “John”; tạo ra một ðối tượng mới thể hiện chuỗi “John”. Các cuộc gọi liên tiếp ðến chuỗi “John” sẽ quy chiếu ðến ðối tượng vừa ðược tạo ra (và không quy chiếu ðến ðối tượng khác của dãy String mà chứa chuỗi “John”; những ðối tượng như vậy không thể ðược tạo lại lần nữa).

Không nên lạm dụng thuộc tính này của ngôn ngữ Java. Phương pháp equals luôn ðược sử dụng ðể so sánh các chuỗi

6.2. Giao diện: nhập vào chuỗi ký tự, số và thông báo

Câu lệnh:

String s = JOptionPane.showInputDialog (“Message”);

ðưa ra một hộp thoại với thông ðiệp “Message” yêu cầu người sử dụng nhập dữ liệu ðầu vào. Sau khi click vào “OK” dòng ký tự nhập vào sẽ ðược lưư lại trong biến thể s. Bằng cách ðóng hộp thoại (hoặc click “cancel”) thì biến thể null ðược trả lại.

Câu lệnh:

JOptionPane.showMessageDialog(null, “Message”);

hiện ra một cửa sổ, dialog với tiêu ðề “Message”.

Ðể sử dụng những dialog câu lệnh import javax.swing.JOptionPane; phải ðược ðưa ra dòng ðầu tiên của chương trình.

Các dialog tạo nên sự tương tác với người sử dụng. Chương trình tiếp theo sẽ chỉ cho thấy làm ðiều ðó như thế nào.

import javax.swing.JOptionPane;
 
public class GreetMsg {
 
  public static void main(String[] args) {
	String name = JOptionPane.showInputDialog("Give your name");
    	if (name == null) 
		name = "";
       	JOptionPane.showMessageDialog( null, "Hello " + name + "!");
    	System.exit(0);
  } 
}

Chương trình hiển thị một cửa sổ yêu cầu dữ liệu vào từ người sử dụng. Nếu chúng ta ðóng cửa sổ mà không nhập tên thì kết quả của phương pháp gọi hàm
JOptionPane.showInputDialog(…) lưu trong biến name sẽ trống. 

Trong trường hợp ðó chúng ta gắn một tham chiếu là chuỗi trống ("").



Lần gọi tiếp theo JOptionPane.showMessageDialog(..) cho thấy một thoại thông ðiệp với tên ðã cho.




Ví dụ thứ hai nối một số các chuỗi ðã cho bởi người sử dụng trong một hộp thoại. Sau ðó sẽ hiện thị tất cả các chuỗi (mỗi chuỗi trên một dòng) trong hộp thoại.

Ðóng hộp thoại sẽ ngắt tập hợp các dữ liệu ðầu vào (các chuỗi) từ người sử dụng. Chương trình không chấp nhận chuỗi trống. Click “OK” mà không có chuỗi cho sắn sẽ dẫn ðến hiển thị một hộp thoại với yêu cầu nhập dữ liệu ðúng (hoặc nhấn “Cancel” ðể hủy). Hình bên hiển thị kết quả của việc chạy chương trình với các tên của ðộng vật cho sẵn như là dữ liệu ðầu vào.

 import javax.swing.JOptionPane;
 
public class ShowStrings {
 
  public static void main(String[] args) {

    String out = "";
    String inp = null;

    while ( (inp = JOptionPane.showInputDialog( "Type in something" )) != null) {
      if (inp.equals("")) 
         JOptionPane.showMessageDialog( null,
					"Input some text or close the dialog"
                                      ); 
      else out = out + '\n' + inp;
    }

    if (out.equals("")) out = "No text was given";
    JOptionPane.showMessageDialog(null, out); 
    System.exit(0);
  } 
}
}

Kết quả:




Chú ý:

    •    biểu thức trong while loop sử dụng một thực tế là giá trị của biến thể chỉ ðịnh bằng với giá trị của ðối số bên trái nó; nó ðược cho vào trong ngoặc bởi vì trật tự thấp hơn          của toán tử chỉ ðịnh =.

    •    các dòng liên tiếp của thông ðiệp ðầu ra ðược tạo ra bằng cách ðính vào dòng mới ký tự ‘\n’
   
    •    các chuỗi ðược so sánh bằng cách sử dụng phương pháp equals()

Cả hai chương trình ðều sử dụng gọi hàm của System.exit(0) ðể ngừng chạy chương trình.
Thông thường một chương trình ngắt sau khi tất cả các câu lệnh trong phương pháp main ðã ðược chạy. Tuy nhiên, ðiều này không ðúng với chương trình có giao diện ðồ họa với người sử dụng (tìm hiểu về vấn ðề này sau). Các hộp thoại là các yếu tố của giao diện ðồ họa với người sử dụng, vì thế các chương trình sử dụng chúng yêu cầu phải ngắt chương trình một cách tường minh với việc gọi System.exit(…).

Ðể ðưa giá trị vào chương trình chúng ta cũng có thể sử dụng lớp Scanner từ gói java.util.

Ví dụ:

import java.util.*;

public class ScanNums {

  public static void main(String[] args) {
    Scanner scan = new Scanner(System.in);
    scan.useLocale(new Locale("en"));
    System.out.println("Số nguyên:");
    int i = scan.nextInt();
    System.out.println("Số thực:");
    double d = scan.nextDouble();
    System.out.println("Ðã ðưa vào: " + i + " " + d);
  }

}

Làm thế nào ðể người sử dụng có thể giới thiệu các con số vào trong một chương trình (sử dụng các dialog)?

Nếu chúng ta viết một số vào trong môi trường vãn bản của một dialog nó sẽ ðược xem như là ðối tượng vãn bản và ở trong chương trình ở dạng ðó. Dĩ nhiên chúng ta không thể tiến hành các thao tác số học trên chuỗi thể hiện con số. Chúng ta phải chuyển ðổi các chuỗi sang dạng mà các số ðược thể hiện trong máy tính.


Một chuỗi thể hiện một con số có thể ðược chuyển sang một số với phương pháp gọi sau:

    aNumber = Integer.parseIn(aString);

trong ðó:

    aNumber – một biến thể thuộc loại int   
    aString – một biểu thức của loại String


Chúng ta xem xét chương trình cộng hai số ðã cho bởi người sử dụng sau như là một ví dụ:

import javax.swing.*;
 
public class ParseInt {
 
  public static void main(String[] args) {

    String s1 = JOptionPane.showInputDialog("Give the first number");
    String s2 = null;
    if (s1 != null) {
      s2 = JOptionPane.showInputDialog("Give the second number");
      if (s2 != null) {
         int l1 = Integer.parseInt(s1);
         int l2 = Integer.parseInt(s2);
         JOptionPane.showMessageDialog(null, "Sum: " + (l1 + l2));
      }
    }

    System.exit(1);
  } 
}
